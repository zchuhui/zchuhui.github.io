AI 编程基础概念

理解 AI 编程的核心范式：从调用 API 到构建智能体，再到工程化落地。

    什么是 AI 编程

    AI 编程（AI-Assisted Programming）是指在软件开发全生命周期中引入大语言模型（LLM）能力，以提升编码、调试、测试、文档和架构设计效率的工程实践。它不仅包含”让 AI 写代码”，更涵盖需求拆解、代码审查、测试生成、性能优化等环节。

    三种核心范式

    1. Copilot 模式（人机协作）

    开发者主导，AI 辅助。典型场景：代码补全、解释代码、生成单测、重构建议。

    • 特点：人在回路，AI 输出作为草稿，由开发者审查采纳。
    • 适用：日常开发、学习新框架、跨语言迁移。

    2. Agent 模式（自主执行）

    AI 主导，按目标自主拆解、执行、验证。典型场景：给定一个 Issue，Agent 自动读代码、改代码、跑测试、提 PR。

    • 特点：AI 拥有工具调用能力（文件读写、Shell、浏览器）。
    • 适用：明确边界的小任务、批量重构、CI 自动修复。

    3. Pipeline 模式（流程编排）

    将 AI 嵌入研发流程的特定环节，如 Code Review、Commit Message 生成、PR 摘要、文档同步。

    • 特点：AI 作为流水线的一环，输入输出确定。
    • 适用：团队工程化、质量门禁、自动化运营。

    关键术语

    术语含义
    Context Window模型一次能处理的输入 token 上限，决定可塞入的代码量
    Token模型处理的最小文本单元，1 个汉字约 1-2 token，1 个英文单词约 1 token
    Embedding把文本映射为向量，用于语义检索和相似度计算
    Tool Calling模型通过结构化输出触发外部工具（函数、API）
    MCPModel Context Protocol，AI 与外部工具/数据源的标准化协议
    RAG检索增强生成，先查知识库再生成回答
    Fine-tuning在基础模型上用领域数据继续训练，定制行为
    Hallucination模型编造不存在的事实，是 AI 编程的主要风险

    AI 编程的能力边界

    擅长

    • 模板化代码生成（CRUD、数据转换、配置文件）
    • 代码解释与文档撰写
    • 单元测试与边界用例发现
    • 跨语言翻译、框架迁移
    • 已知模式的 bug 定位

    不擅长

    • 复杂业务逻辑的自主设计（缺乏全局上下文）
    • 涉及隐性领域知识的判断
    • 性能敏感的底层优化
    • 跨多服务的分布式一致性设计
    • 创新性架构方案

    工程化核心挑战

    1. 上下文管理：如何把相关代码、文档、规范精准送入模型？
    2. 质量保障：AI 生成的代码如何自动校验（类型、测试、Lint）？
    3. 成本控制：Token 消耗如何计量与优化？
    4. 安全合规：代码隐私、密钥泄漏、许可证风险如何防范？
    5. 可观测性：AI 调用链如何追踪、复盘、持续改进？
