## 从想法到产品

作为一名 AI Builder，我相信**最好的学习方式就是构建**。这篇文章记录我把一个模糊想法变成可用产品的完整流程。

## 我的工作流

1. **定义问题**：先想清楚"为谁解决什么"，而不是先想用什么模型。
2. **最小验证**：用最简单的 prompt + 现成模型跑通核心链路。
3. **迭代评测**：建立小而稳的评测集，用数据驱动优化。
4. **工程化**：缓存、降级、可观测性，一个都不能少。

## 一些心法

> 模型会变，但对问题的理解和对用户的同理心不会过时。

- 不要过早优化模型，先把产品闭环跑通。
- Prompt 是接口，要像写 API 一样认真对待。
- 评测比直觉可靠，但别让评测绑架了产品直觉。
